Security:
 - use HttpOnly for cookies storing sensitive information (prevents client side script accessing the protected cookie)
 - Refresh tokens should be hashed in database (like a password)
 - Message Authentication Code (MAC)

Cookie Storage:
 - HttpOnly (No scripts can access it)
 - SameSite = strict (No cross-origin  cookie sharing)
 - secure (SSl only)
